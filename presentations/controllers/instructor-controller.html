🏗️ Architecture Overview

Instructor endpoints require INSTRUCTOR or ADMIN role. Most also perform an ownership check — only the course instructor (or an admin) can modify a course, its lessons, quiz, or view its enrollments.

Request → authenticate → requireRole('INSTRUCTOR', 'ADMIN') → Ownership check → Instructor Controller

// Ownership guard — reused across course endpoints const course = await getCourseIfOwner(id, req.user.userId, req.user.role); if (!course) return res.status(404).json({ error: 'Course not found' }); // Re-slugify only if title actually changed if (data.title && data.title !== course.title) { let slug = slugify(data.title); const existing = await prisma.course.findUnique({ where: { slug } }); if (existing && existing.id !== course.id) slug = uniqueSlug(slug); data.slug = slug; }

PATCH /api/courses/:courseId/lessons/reorder Transactional

Accepts an ordered array of lesson IDs. All updates run in a single database transaction.

// Validates all IDs belong to the course first const existingLessons = await prisma.lesson.count({ where: { id: { in: lessonIds }, courseId }, }); if (existingLessons !== lessonIds.length) { return res.status(400).json({ error: 'Some lesson IDs do not belong to this course' }); } // Transaction — all updates or none await prisma.$transaction( lessonIds.map((id, index) => prisma.lesson.update({ where: { id }, data: { order: index } }) ) );
📌 Request Body
{ "lessonIds": ["uuid-1", "uuid-2", "uuid-3"] } — the array index becomes the new order value.

📝 Quiz Management

POST /api/lessons/:lessonId/quiz One per lesson

Each lesson can have at most one quiz. Creating a second returns 409 Conflict. Ownership is verified through the lesson → course chain.

// Ownership check traverses: lesson → course → instructorId const lesson = await getLessonIfCourseOwner(lessonId, req.user.userId, req.user.role); if (!lesson) return res.status(404).json({ error: 'Lesson not found' }); const existing = await prisma.quiz.findUnique({ where: { lessonId } }); if (existing) return res.status(409).json({ error: 'Lesson already has a quiz' });

Ownership Chains

Quiz and question ownership checks traverse up through the model hierarchy to reach the course instructor:

// Quiz ownership: quiz → lesson → course → instructorId async function getQuizIfCourseOwner(quizId, userId, role) { const quiz = await prisma.quiz.findUnique({ where: { id: quizId }, include: { lesson: { include: { course: { select: { instructorId } } } } }, }); if (!quiz) return null; if (role === 'ADMIN') return quiz; // Admins bypass ownership if (quiz.lesson.course.instructorId !== userId) return null; return quiz; } // Question ownership: question → quiz → lesson → course → instructorId async function getQuestionIfCourseOwner(questionId, userId, role) { ... }

🔐 Ownership Check Patterns

Four helper functions implement the ownership check pattern. Each traverses the Prisma relation chain to find the course's instructorId, then compares it to the authenticated user.

HelperChainUsed By
getCourseIfOwner direct: course.instructorId Course CRUD, publish, stats, lesson CRUD, enrollments
getLessonIfCourseOwner lesson → course.instructorId Create quiz for lesson
getQuizIfCourseOwner quiz → lesson → course.instructorId Update/delete quiz, add question
getQuestionIfCourseOwner question → quiz → lesson → course.instructorId Update/delete question

All helpers follow the same pattern: load the resource with nested includes, return null if not found, bypass if role === 'ADMIN', then compare instructorId to userId.
This means ADMINS can manage any instructor's content — no special admin course endpoints needed beyond the feature toggle.
