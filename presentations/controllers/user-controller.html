🏗️ Architecture Overview

NextAuth.js runs in the Next.js frontend and handles sessions. The Express backend provides data APIs and validates NextAuth-issued JWTs.

NextAuth → Express /api/auth/login → JWT (HS256) → NextAuth session cookie
Server Action → Read cookie → Bearer → Express /api/* → JSON Response

🔁 Authentication Flow

Login Flow

// 1. User submits credentials in Next.js form // 2. NextAuth Credentials provider calls Express: async authorize(credentials) { const res = await fetch(`${BACKEND_URL}/api/auth/login`, { method: 'POST', body: JSON.stringify(credentials), }); if (!res.ok) return null; return await res.json(); // → { id, name, email, role } } // 3. NextAuth creates JWT (HS256, 7d expiry) // 4. JWT stored in httpOnly cookie (authjs.session-token) // 5. JWT callback stores userId + role in token jwt({ token, user }) { if (user) { token.sub = user.id; token.role = user.role; } return token; }

Signup Flow

// 1. Server Action calls Express to create user const res = await fetch(`${BACKEND_URL}/api/auth/signup`, { method: 'POST', body: JSON.stringify({ name, email, password, role }), }); // Express returns user data (no JWT — NextAuth handles that) // 2. If signup succeeds, trigger NextAuth sign-in: await signIn('credentials', { email, password, redirect: false }); // → authorize() calls Express /api/auth/login → NextAuth creates session

API Call Flow (Frontend → Express)

// Server Component / Server Action reads auth cookie via next/headers // and sends the raw JWT as Bearer token to Express: export async function apiFetch(path, options = {}) { const token = (await cookies()).get('authjs.session-token')?.value; const headers = new Headers(options.headers); if (token) headers.set('Authorization', `Bearer ${token}`); return fetch(`${BACKEND_URL}${path}`, { ...options, headers }); } // Express middleware verifies the JWT with jsonwebtoken (HS256): const payload = jwt.verify(token, secret, { algorithms: ['HS256'] }); req.user = { userId: payload.sub, role: payload.role };

⚙️ NextAuth.js Configuration

frontend/src/lib/auth.ts — central auth config used across the Next.js app.

JWT Strategy

// Custom encode/decode overrides NextAuth's default (jose + HKDF) // Uses simple HS256 with the raw secret — compatible with backend's jsonwebtoken jwt: { async encode({ token, secret }) { return new SignJWT(token) .setProtectedHeader({ alg: 'HS256' }) .setIssuedAt() .setExpirationTime('7d') .sign(new TextEncoder().encode(secret)); }, async decode({ token, secret }) { const { payload } = await jwtVerify(token, new TextEncoder().encode(secret), { algorithms: ['HS256'], }); return payload; }, }

Backend JWT Verification

// backend/src/middleware/auth.ts // Uses jsonwebtoken to verify NextAuth-issued JWTs: export function authenticate(req, res, next) { const token = req.headers.authorization?.slice(7); // jwt.verify with HS256 — same secret, same algorithm const payload = jwt.verify(token, process.env.AUTH_SECRET, { algorithms: ['HS256'], }); req.user = { userId: payload.sub, role: payload.role }; next(); }
