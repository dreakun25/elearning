🏗️ Architecture Overview

All admin endpoints sit behind a middleware chain of authenticate → requireRole('ADMIN'). Only users with the ADMIN role can access any endpoint in this controller.

Request → authenticate → requireRole('ADMIN') → Admin Controller → JSON / CSV

📊 Stats & Course Management

GET /api/admin/stats

Returns platform-wide aggregates computed via three parallel Prisma queries.

// Three queries run in parallel with Promise.all const [usersByRole, coursesByStatus, totalEnrollments] = await Promise.all([ prisma.user.groupBy({ by: ['role'], _count: { id: true } }), prisma.course.groupBy({ by: ['published'], _count: { id: true } }), prisma.enrollment.count(), ]);

GET /api/admin/courses Paginated

Unlike the public courses endpoint, this returns all courses including drafts and unpublished. Includes instructor info and counts of lessons, enrollments, and reviews.

// Includes instructor details + relation counts const courses = await prisma.course.findMany({ where, skip, take: limit, orderBy: { createdAt: 'desc' }, include: { instructor: { select: { id: true, name: true, email: true } }, _count: { select: { lessons: true, enrollments: true, reviews: true } }, }, });

PATCH /api/admin/courses/:id/feature

Toggles a course's featured boolean. Frontend can use this to surface promoted courses on the landing page.

// Toggle — no request body needed const course = await prisma.course.findUnique({ where: { id } }); if (!course) return res.status(404).json({ error: 'Course not found' }); const updated = await prisma.course.update({ where: { id }, data: { featured: !course.featured }, select: { id, title, slug, featured, published }, });

🧩 Controller Patterns

Zod Validation

All mutation endpoints use safeParse for input validation, consistent with the existing user controller pattern.

// Standard validation pattern used across all admin endpoints const parsed = updateRoleSchema.safeParse(req.body); if (!parsed.success) { return res.status(400).json({ error: 'Validation failed', details: parsed.error.flatten().fieldErrors, }); }

Self-Operation Guards

Admins cannot demote or delete themselves — prevents accidental lockout.

// Guard in updateUserRole if (id === req.user.userId) { return res.status(400).json({ error: 'Cannot change your own role' }); } // Guard in deleteUser if (id === req.user.userId) { return res.status(400).json({ error: 'Cannot delete your own account' }); }

Parallel Queries

Where possible, independent Prisma queries run in parallel via Promise.all for performance.

// Paginated list + total count — runs simultaneously const [users, total] = await Promise.all([ prisma.user.findMany({ ... }), prisma.user.count({ where }), ]); // Stats endpoint — three aggregates in parallel const [usersByRole, coursesByStatus, totalEnrollments] = await Promise.all([ prisma.user.groupBy({ ... }), prisma.course.groupBy({ ... }), prisma.enrollment.count(), ]);
